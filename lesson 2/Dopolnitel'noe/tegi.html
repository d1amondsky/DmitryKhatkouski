  1. Первым делом в нашем списке идет тег ‹dl›, входящий в тройку тегов ‹dl›, ‹dt›, ‹dd›.
    Теги предназначены для создания списка определений. Список начинается с контейнера ‹dl›, куда входит тег ‹dt›, создающий термин и тег ‹dd›, созающий описание данного термина.

    Вот пример создания такого списка:

    <dl>
    <dt>GIF</dt>
    <dd>Формат графических файлов, широко применяемый при создании сайтов.
    GIF использует 8-битовый цвет и эффективно сжимает сплошные цветные области, при этом сохраняя детали изображения.</dd>
    <dt>JPEG</dt>
    <dd>Популярный формат графических файлов, широко применямый при создании сайтов и хранения изображений. JPEG поддерживает 24-битовый цвет и сохраняет яркость и оттенки цветов в фотографиях. Данный формат называют сжатием с потерями, поскольку алгоритм JPEG выборочно отвергает данные. Метод сжатия может исказить деталь в рисунке, особенно содержащий текст или изображение с четкими краями. Формат JPEG не поддерживает прозрачность, когда вы сохраняете фотографию в формате JPEG, прозрачные пиксели заполняются определенным цветом.</dd>
    </dl>

    Вот как этот пример будет выглядеть на странице:

    GIF
    Формат графических файлов, широко применяемый при создании сайтов. GIF использует 8-битовый цвет и эффективно сжимает сплошные цветные области, при этом сохраняя детали изображения.
    JPEG
    Популярный формат графических файлов, широко применямый при создании сайтов и хранения изображений. JPEG поддерживает 24-битовый цвет и сохраняет яркость и оттенки цветов в фотографиях. Данный формат называют сжатием с потерями, поскольку алгоритм JPEG выборочно отвергает данные. Метод сжатия может исказить деталь в рисунке, особенно содержащий текст или изображение с четкими краями. Формат JPEG не поддерживает прозрачность, когда вы сохраняете фотографию в формате JPEG, прозрачные пиксели заполняются определенным цветом.


  2. Вторым в нашем списке идет тег ‹pre›. Этот тег определяет блок предварительно форматированного текста. Такой текст отображается обычно моноширинным шрифтом и со всеми пробелами между словами. По умолчанию, любое количество пробелов идущих в коде подряд, на веб-странице показывается как один. Тег ‹pre› позволяет обойти эту особенность и отображать текст как требуется разработчику. В отличие от тега ‹plaintext›, использование которого осуждается в HTML 4, внутри контейнера ‹pre› допустимо применять любые теги кроме следующих: ‹big›, ‹img›, ‹object›, ‹small›, ‹sub›, ‹sup›.

    Пример:

    <pre> 
                   -                -----   
    
	       -----           |----- 
    
	      ||----|          ----||  
    
	      ||-----         -----||  
    
	      ||-----|       |-----|| 
    
	      || -----       ------|| 
    
	      ||- ----|     |------|| 
    
	      ||---||--     -------|| 
    
	      ||--|| --|   |-------|| 
    
	      || -|| |--   --- - --|| 
    
	      || -||  --|-|--| - ---| 
    
	      |---||  |-----| |-----| 
    
	      |---||   |----  |-----| 
    
	      |----|    ---   |-----| 
    
	      |-----          ------|
    </pre>

  3. Тег ‹code› предназначен для отображения одной или нескольких строк текста, который представляет собой программный код. Сюда относятся имена переменных, ключевые слова, тексты функции и т.д. Браузеры обычно отображают содержимое контейнера ‹code› как моноширинный текст уменьшенного размера. В отличие от тега ‹pre› дополнительные пробелы внутри контейнера ‹code› не учитываются, так же, как и переносы текста. Поэтому используйте тег ‹br› или ‹p› для создания переносов.

    Вот пример написания кода с нашим тегом:

    <p>Код программы</p>
    <p><code>
    function checkParent (src, dest) {<Br>
    while (src != null) {<Br>
    if (src.tagName == dest) return src<Br>
    src = src.parentElement<Br>
    }<Br>
    return null<Br>
    }</code></p>

    Вот так данный код будет выглядеть на странице браузера:

    Код программы

    function checkParent (src, dest) {
    while (src != null) {
    if (src.tagName == dest) return src
    src = src.parentElement
    }
    return null
    }


  4. Тег ‹tt› отображает текст моноширинным текстом. Этот тег относится к группе тегов физического форматирования.

    Пример кода:

    <p><tt>Lorem ipsum dolor sit amet</tt></p>
    <p>Lorem ipsum dolor sit amet</p>

    Сраните как выглядит это в итоге:

    Lorem ipsum dolor sit amet

    Lorem ipsum dolor sit amet


  5. Тег ‹kbd› используется для обозначения текста, который набирается на клавиатуре или для названия клавиш. Браузеры обычно помечают текст в контейнере ‹kbd› моноширинным шрифтом.

    Пример кода:

    <p>Для быстрого закрашивания выделенной области цветом переднего плана
    используйте комбинацию клавиш <kbd> &lt;Alt&gt;+&lt;Backspace&gt; </kbd>
    или <kbd> &lt;Alt&gt;+&lt;Delete&gt; </kbd> </p>

    Результат приведен ниже:

  6. Тег ‹var› используется для выделения переменных компьютерных программ. Браузеры обычно помечают текст в теге ‹var› курсивным начертанием.

    Давайте посмотрим на пример кода:

    <p>В данном примере делается запрос к базе данных для получения поля <var>content_title</var>
    из таблицы <var>content</var>, причем должно удовлетворяться условие
    <code>section_id = 1</code>.</p>

  7. Следующими идут теги ‹sup› и ‹sub›.
    ‹sup› отображает шрифт в виде верхнего индекса. Шрифт при этом отображается выше базовой линии текста и уменьшенного размера.
    ‹sub› отображает шрифт в виде нижнего индекса. Текст при этом располагается ниже базовой линии остальных символов строки и уменьшенного размера.

    Пример кода:

    СН<sub>3</sub>СНО + NH<sub>2</sub>OH = CH<sub>3</sub>C(=NOH)H + H<sub>2</sub>O
    X <sup>n</sup> + Y <sup>n</sup> = Z <sup>n</sup>

    Итог:

    СН3СНО + NH2OH = CH3C(=NOH)H + H2O
    X n + Y n = Z n


  8. Тег ‹q› используется для выделения в тексте цитат. Содержимое контейнера автоматически отображается в браузере в кавычках.

    Пример кода:

    Станислав Лец утверждал: <q>Чаще всего выход там, где был вход</q>.

    Отображение на странице:

    Станислав Лец утверждал: Чаще всего выход там, где был вход.


  9. Тег ‹ins› предназначен для выделения текста, который был добавлен в новую версию документа. Подобное форматирование позволяет отследить, какие изменения в тексте документа были сделаны. Браузеры обычно помечают текст в контейнере ‹del› как подчеркнутый.
    Тег ‹del› используется для выделения текста, который был удален в новой версии документа. Подобное форматирование позволяет отследить, какие изменения в тексте документа были сделаны. Браузеры обычно помечают текст в контейнере ‹del› как перечеркнутый.
